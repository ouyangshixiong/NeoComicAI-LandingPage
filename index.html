工作方式How it works

8 个步骤，把零散灵感变成可沟通、可制作的分镜与视频流水线。Eight steps to turn loose ideas into a production-ready storyboard and video pipeline.

  1. 1. 创建作品1. Create a project

    输入标题与简介，建立项目空间，为后续剧本、分镜与素材提供统一归档。Enter a title and summary to create a dedicated space for scripts, boards and assets.

  2. 2. 故事输入2. Input the story

    粘贴原始故事或创意大纲，也可以从已有文档中复制内容。Paste the original story, outline or copy from existing documents.

  3. 3. 剧情改写3. Rewrite the script

    调用 LLM 生成结构化剧本，并由创作者人工校对、调整节奏与对白。Use LLMs to generate a structured script, then refine pacing and dialogue by hand.

  4. 4. 分镜切分4. Panelize

    把剧本拆解为镜头组与分镜面板，为每一镜头补充景别、构图与动作信息。Break the script into shot groups and panels, adding shot size, framing and action notes.

  5. 5. 建立角色库5. Build the character library

    为关键角色补充设定与参考图，让后续分镜图与视频生成的人物更稳定统一。Enrich key characters with profiles and reference images so later boards and videos stay consistent.

  6. 6. 生成九宫格分镜图6. Generate 3×3 tiles

    生成统一风格的九宫格分镜图，用于对齐镜头设计并作为视频生成参考帧。Generate unified 3×3 storyboard tiles to align shot design and serve as video reference frames.

  7. 7. 生成分镜视频7. Generate shot videos

    按镜头或分片生成视频片段，快速验证节奏与画面表达。Generate clips per shot or tile to quickly validate rhythm and visual storytelling.

  8. 8. 合并与导出8. Merge and export

    将多个片段合并为完整分镜视频，导出用于剪辑、提案或团队沟通。Merge clips into a complete animatic and export it for editing, pitching or team reviews.

常见问题Frequently asked questions

Q1：它能把什么输入变成什么输出？Q1: What inputs and outputs does it support?

输入是文本故事或创意大纲；输出是可编辑剧本、分镜面板（含场景描述）、九宫格分镜图，以及可合并的分镜视频片段。Inputs are text stories or outlines; outputs include editable scripts, panel lists with scene descriptions, 3×3 storyboard tiles and mergeable shot video clips.

Q2：如何保证角色一致性？Q2: How do you keep characters consistent?

通过角色库管理角色设定与参考图，在分镜与视频生成时作为上下文条件输入，尽量减少人物造型与气质的漂移。A character library stores profiles and reference images, which are injected as context into panel and video generation to reduce drift in appearance and tone.

Q3：可以不用视频功能只做漫画分镜吗？Q3: Can I only use it for comic-style storyboards without video?

可以。分镜图生成、角色库与工作流管理本身就是独立价值，视频相关节点可以按需启用或跳过。Yes. Storyboard generation, the character library and workflow management stand on their own; video nodes are optional.

Q4：支持哪些图像与视频线路？Q4: Which image and video providers are supported?

系统提供可扩展的 Provider 结构，并已接入多种图像与视频生成线路，支持按项目或用户配置切换与回退。The system uses an extensible provider structure and already integrates multiple image and video routes, with per-project or per-user configuration and fallbacks.

Q5：数据与素材存在哪里？Q5: Where are data and assets stored?

默认存储在本地数据目录，包括 SQLite 数据库与图片、视频文件，也可以对接首尾帧云服务同步参考帧到公网可访问的静态文件服务。By default, everything is stored locally in a SQLite database and media directories. You can also hook into a keyframe cloud service to sync reference frames to public static hosting.

Q6：适合团队协作吗？Q6: Is it suitable for team collaboration?

适合。作品、角色、风格、分镜与素材都按结构化对象管理，便于复用与规范化流程，也提供 Agent API 便于接入内部系统。Yes. Projects, characters, styles, panels and media are all managed as structured objects for reuse and standardization, and the Agent API makes it easy to integrate into internal systems.
